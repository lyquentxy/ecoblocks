# Capability System

Capabilities describe what the Hub can do with devices and services.

For now, keep capabilities simple and close to the mock agent flow. Use `agent/capabilities/` when a capability becomes reusable across UI, runtime, or examples.

Examples:

- Read a temperature-like value.
- Set a fan-like actuator state.
- Report agent/device profile status to Blockly.

Avoid detailed hardware schemas until protocol work settles.
