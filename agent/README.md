# Agent

Hub-side intelligence and device-facing modules.

- `core/`: pure Dart package shared with the Flutter app.
- `runtime/`: future runtime orchestration beyond the current small core package.
- `capabilities/`: future reusable device/service abilities.
- `devices/`: hardware node notes and references.
- `protocols/`: bus protocol notes and experiments.
- `embedded/`: low-memory or legacy Android route.
- `firmware/`: future firmware sources.
- `safety/`: future Hub-side safety rules.
- `memory/`: future low-memory strategy and profiling notes.

Keep mock data minimal until hardware protocol choices are proven.
